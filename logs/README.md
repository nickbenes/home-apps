# Logs Directory

This folder contains logs and instructions for each session with relevant context.

## Directory Structure
- `sessions/`: Tracks session logs with timestamps (e.g., `session-YYYY-MM-DD-HHMM.md`).
- `context/`: Holds persistent context files summarizing key decisions, architecture details, and goals.
- `agent/`: Contains any specific instructions for AI agents or collaborators.

## Naming Conventions
- **Session Logs**: `session-YYYY-MM-DD-HHMM.md`
- **Context Files**: Named by subject (e.g., `architecture.md`, `workflow-policies.md`).
- **Agent Instructions**: Use task-based names.

These structures help persist information across platforms and sessions, especially when switching environments.
